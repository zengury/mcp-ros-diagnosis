## 适用场景

- 关节响应迟缓，动作跟随误差大
- 运动时出现明显抖动或持续振荡
- 步态中某关节"软"或"过硬"
- 更换减速器/电机后需要重新整定参数
- 初始部署时批量调参

---

## 核心概念：为什么用 AI 调参？

传统手动调参是"试→观察→猜→再试"的线性过程，效率低且经验难以复用。

本系统实现了一个**自主研究闭环**（受 AutoResearch 框架启发）：

```
┌─────────────────────────────────────────────────────┐
│               AI-PID 调参闭环                        │
│                                                     │
│  [LLM 分析历史波形] → [提出参数假设]                  │
│          ↑                    ↓                     │
│  [存入历史 + 评分]     [安全围栏校验]                  │
│          ↑                    ↓                     │
│  [计算多维度指标] ← [执行阶跃响应实验]                  │
└─────────────────────────────────────────────────────┘
```

**LLM 的作用不只是"改数字"**，它会阅读诊断文字（超调多少、调节多慢、是否振荡）并给出**工程依据**，类似有经验工程师的思考过程。

---

## 三大工程挑战及解决方案

### 挑战 1：评价函数量化（怎样让 Agent 读懂波形）

**解决方案**：`scorer.py` 将时间序列转换为 5 个控制理论指标：

| 指标 | 含义 | 满分条件 |
|------|------|----------|
| 超调量 (Overshoot) | 峰值超出目标 % | ≤ 0% |
| 上升时间 (Rise Time) | 10%→90% 用时 | ≤ 0.3s |
| 调节时间 (Settling Time) | 最后离开 ±2% 带 | ≤ 0.5s |
| 稳态误差 (SSE) | 稳定后偏差 % | ≤ 0.5% |
| 振荡次数 | 误差过零次数 | ≤ 2 次 |

综合分 = 100 - 各维度扣分（最低 0 分），并输出**中文诊断文字**。

### 挑战 2：安全围栏（防止极端参数损坏硬件）

**三层防护**（`safety.py`）：

1. **静态边界**：Kp/Ki/Kd 参数提交时立即拒绝超界值（来自 `robot_schema.yaml:pid_safety_bounds`）
2. **实验前检查**：温度 < 60°C、电量 > 20%、无通信丢失
3. **实验中监控**：力矩超限 → 立即中止 → 发送零力矩命令

### 挑战 3：环境一致性（保证实验可比性）

每次实验自动记录**环境快照**：

```json
{
  "timestamp": 1709123456.7,
  "battery_soc_pct": 78.5,
  "battery_v": 47.2,
  "joint_temp_c": 38.1,
  "comm_lost": 0
}
```

历史记录中的每条实验都携带此快照，LLM 可以识别"这次电量特别低导致结果偏差"。

## 调参工作流

### 标准流程（推荐）

```
1. pid_safety_check("left_knee")
   → 确认 ready_to_tune = true

2. pid_run_auto_tuning("left_knee", max_iterations=20, target_score=85)
   → 等待自动迭代完成

3. pid_get_best("left_knee")
   → 获取最终推荐参数
```

### 手动精细调参流程

```
1. pid_safety_check("left_knee")          # 检查环境

2. pid_run_experiment("left_knee",        # 首次测试当前参数
       kp=10.0, ki=0.5, kd=2.0)

3. pid_propose_params("left_knee",        # 让 LLM 建议下一组
       current_kp=10.0, current_ki=0.5, current_kd=2.0,
       current_score=62.0)

4. # 根据建议执行实验，观察评分变化，循环直到满意

5. pid_get_history("left_knee")           # 查看全程记录

6. pid_get_best("left_knee")              # 提取最优参数
```

## 分数诊断指南

| 分数 | 等级 | 含义 | 典型问题 |
|------|------|------|----------|
| 90-100 | A | 优秀 | 可直接投入使用 |
| 75-89 | B | 良好 | 适合一般任务 |
| 60-74 | C | 合格 | 精细操作可能不稳 |
| 40-59 | D | 较差 | 明显跟随误差或振荡 |
| 0-39 | F | 失败 | 系统不稳定，禁止使用 |

### 常见波形问题诊断

**超调 > 20%（Kp 过大 / Kd 不足）**：
- 减小 Kp 至 0.8 倍
- 增大 Kd（约为当前值的 1.3 倍）

**响应慢（上升时间 > 1.0s）**：
- 增大 Kp（约 1.3 倍）
- Ki 维持或略减（避免积分饱和拖慢响应）

**稳态误差 > 3%（Ki 不足）**：
- 增大 Ki（每次 +0.05）
- 配合适当 Kd 避免引入振荡

**持续振荡（过零次数 > 10）**：
- 大幅减小 Kp（0.6 倍）
- 增大 Kd
- 降低 Ki（积分饱和会加剧振荡）

## 各关节组推荐初始参数范围

| 关节组 | Kp 初始 | Ki 初始 | Kd 初始 | 备注 |
|--------|---------|---------|---------|------|
| 腿部（膝、髋） | 15-25 | 0.5-1.0 | 3-6 | 惯量大，需要更强 P |
| 踝关节 | 8-15 | 0.2-0.5 | 2-4 | 地面接触，阻尼敏感 |
| 腰部 | 10-20 | 0.3-0.8 | 2-5 | 影响全身平衡 |
| 手臂（肩、肘） | 5-15 | 0.1-0.5 | 1-3 | 惯量小，易振荡 |
| 手腕 | 3-8 | 0.05-0.2 | 0.5-2 | 最轻，Kd 要保守 |

---

## 安全注意事项

1. **真机调参前必须**：机器人固定在测试台，关节活动方向无障碍物
2. **禁止**在机器人行走状态下调参（仅支持单关节测试台模式）
3. **每次实验后**：观察温度，超过 55°C 停止，冷却 5 分钟后继续
4. **Mock 模式验证**：新的参数范围必须先在仿真模式下验证无 F 级结果
